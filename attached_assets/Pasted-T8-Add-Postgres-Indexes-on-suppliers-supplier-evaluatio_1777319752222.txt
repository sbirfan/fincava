T8 — Add Postgres Indexes on suppliers + supplier_evaluations (Idx)

Task ID: Idx
Type: TDD Hardening — Performance
Priority: MEDIUM — parallelizable with T4
Estimated effort: 1–2 hours
Dependency: None

---

OBJECTIVE

Add two Postgres indexes to improve admin query performance as the supplier count grows:

1. suppliers(sellable_status) — used by admin suppliers list queries filtered by status
2. supplier_evaluations(supplier_id, created_at DESC) — used by history lookups and the graduation pipeline

At current scale these queries use sequential scans. As supplier count grows (target: ≥50 SELLABLE for Phase 2 matching), without indexes these queries degrade linearly.

Execute any time. Index creation is non-destructive and safe for concurrent reads.

CONFLICT DETECTION

---

PROPOSED APPROACH

Add index definitions directly in the Drizzle schema files using Drizzle’s index() function, then generate and apply the migration.

This keeps indexes under version control and schema management.

---

ALTERNATIVES

Alternative | Reason Rejected
Add indexes via raw SQL only | Bypasses Drizzle schema — indexes disappear on schema reset
Add composite index (sellable_status, id) | Premature optimization — single column sufficient at current scale
Defer to Phase 2 | Risk of slow admin queries when real suppliers start accumulating

---

IMPLEMENTATION

## STEP 1: Add index to suppliers schema

In src/db/schema/suppliers.ts:

import { pgTable, ..., index } from 'drizzle-orm/pg-core';

export const suppliersTable = pgTable('suppliers', {
// ... existing columns unchanged ...
}, (table) => ({
sellableStatusIdx: index('suppliers_sellable_status_idx').on(table.sellableStatus),
}));

---

## STEP 2: Add composite index to supplier_evaluations schema

In src/db/schema/supplier-evaluations.ts:

export const supplierEvaluationsTable = pgTable('supplier_evaluations', {
// ... existing columns unchanged ...
}, (table) => ({
supplierIdCreatedAtIdx: index('supplier_evaluations_supplier_id_created_at_idx')
.on(table.supplierId, table.createdAt.desc()),
}));

---

## STEP 3: Generate and apply migration

pnpm drizzle-kit generate
pnpm drizzle-kit migrate
