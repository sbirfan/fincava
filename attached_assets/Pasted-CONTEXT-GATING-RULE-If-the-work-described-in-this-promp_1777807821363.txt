FORM STRUCTURE — /buyer/onboarding

Overall pattern:
- 4 active sections rendered as steps with a progress indicator (Step 1 of 4, Step 2 of 4…)
- Section navigation: Next / Back buttons. Final section has Submit instead of Next.
- Conditional questions only render when their parent category is selected (see logic below)
- On each Next: PATCH /api/buyer/onboarding with current section's data before advancing
- On load: GET /api/buyer/onboarding — if profile exists, pre-fill all fields. If 404, start fresh.
- On final Submit: PATCH last section data → then fire matching trigger (Change 4) → redirect to /dashboard/matches with toast "Profile complete — your supplier matches are ready."
