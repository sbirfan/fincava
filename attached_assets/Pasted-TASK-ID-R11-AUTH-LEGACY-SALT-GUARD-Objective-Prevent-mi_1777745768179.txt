TASK ID: R11-AUTH-LEGACY-SALT-GUARD

Objective:
Prevent missing LEGACY_HASH_SALT from causing uncontrolled auth request failures while keeping the insecure hardcoded fallback removed.

Problem:
R2 correctly removed the hardcoded legacy salt fallback, but legacyHash() currently throws during request-time password verification if LEGACY_HASH_SALT is missing. If a legacy SHA-256 user logs in while the env var is missing, auth may fail with an uncontrolled 500.
