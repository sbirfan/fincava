IMPORTANT CONTEXT

Original intended purpose:

Origin Stories was meant to be a lightweight public farm/farmer discovery layer.

Basic intended flow:

Admin or AI/public-data ingestion creates an Origin Story
→ it appears on /origin-stories
→ user clicks tile
→ user lands on an Origin Story / Farm Detail page
→ if linked supplier exists, user may go to Supplier Profile / Products
→ if farmer has not claimed the profile, show Claim Your Profile
→ if no linked supplier exists, show no supplier/profile button

Known entry points:
1. AI/public web-search ingestion
2. Admin-created story from admin console

Unknown:
- whether other entry points exist
- whether current origin_stories data is actually used anywhere important
- whether prior product-linked design is live, dead, partial, or legacy
- whether existing complexity is real dependency or abandoned drift

A+B navigation/context changes are already completed.
This audit is NOT about implementing new behavior.
