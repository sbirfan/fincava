FEATURE 2 — Editable sourcing profile at /dashboard/profile

Layout:
- Single page at /dashboard/profile, rendered inside DashboardLayout (not AppLayout — this is a dashboard sub-page)
- Left column (main, ~70% width): four collapsible section cards
- Right column (sidebar, ~30% width, sticky): completion overview

Sticky sidebar content:
- Overall completion % (large number, color-coded: <50% red, 50–79% amber, 80–99% blue, 100% green)
- Per-section status list:
  Section 1 — Company Profile: Complete ✅ / In Progress 🔄 / Empty ○
  Section 2 — Product Interests: [status]
  Section 3 — Volume & Pricing: [status]
  Section 4 — Quality & Values: [status]
- Clicking a section status label scrolls to and expands that section

Section cards — default state:
- First incomplete section: expanded by default on page load
- All other sections: collapsed by default
- Each card has a header with section name + status badge + chevron toggle

Section card content:
Each section renders all its fields in editable form using shadcn/ui components (Select, Checkbox, RadioGroup, etc.) — same field definitions as the wizard.
Same conditional rendering rules apply:
  Coffee questions (Q6/Q7/Q12/Q21) only shown if COFFEE in targetProducts
  Cacao questions (Q8/Q13/Q22) only shown if CACAO in targetProducts
  Fruit questions (Q9/Q14) only shown if EXOTIC_FRUIT in targetProducts
  Q24 only shown if buyerSegment is specialty_roaster or craft_chocolatier
  Q32 only shown if sustainabilityImportance ≠ not_important

Read-only display at top of each relevant section (do not re-ask these):
  Section 1 top: "Company: [companyName], [country]"
  Section 2 top: "Sourcing categories: [targetProducts list]"
  Section 4 top: "Required certifications: [requiredCertsP1 list]"

Auto-save behavior:
- Auto-save fires on field blur (when user leaves a field) or on select/checkbox change
- Debounce: 500ms after the triggering event
- Do NOT fire while a dropdown is actively open
- On save: call PATCH /api/buyer/onboarding with the changed section's fields only
- After successful save: show "Saved ✓" indicator on the section card header for 2 seconds, then fade
- On save error: show "Save failed — retry" with a manual retry button
- No explicit Submit button anywhere on this page

Section completion logic (mirrors backend):
A section is "Complete" when all non-conditional required fields in that section have a non-null value.
A section is "In Progress" when at least one field has a value but not all required fields are filled.
A section is "Empty" when no fields in that section have a value.

Data loading:
On page mount: GET /api/buyer/onboarding
If 200: pre-fill all fields, compute section statuses client-side
If 404: show empty form with a banner "Start your sourcing profile to get matched with suppliers. [Start Profile →] links to /buyer/onboarding"
Show a loading skeleton while the GET is in flight

Fields per section — use exact same field names as the wizard (BL-1 inherits from P1.5-B):

SECTION 1 — Company Profile
  buyerSegment (single select)
  locationCount (single select)
  annualBudgetUsd (single select)

SECTION 2 — Product Interests
  coffeeQualityTier (single select, conditional: COFFEE)
  coffeeFlavorProfile (multi-select, conditional: COFFEE)
  cacaoFlavorProfile (single select, conditional: CACAO)
  fruitForm (multi-select, conditional: EXOTIC_FRUIT)
  availabilityRequirement (single select)
  orderFrequency (single select)

SECTION 3 — Volume & Pricing
  coffeeOrderSizeKg (single select, conditional: COFFEE)
  cacaoOrderSizeKg (single select, conditional: CACAO)
  fruitOrderSizeKg (single select, conditional: EXOTIC_FRUIT)
  priceSensitivity (single select)
  priceTransparency (multi-select)

SECTION 4 — Quality & Values
  certsNiceToHave (multi-select)
  traceabilityLevel (single select)
  qualityDocRequired (multi-select)
  coffeeDefectRate (single select, conditional: COFFEE)
  cacaoMoldPct (single select, conditional: CACAO)
  sourceConsistency (single select)
  qualityVerification (multi-select, conditional: specialty_roaster or craft_chocolatier)
  sustainabilityImportance (single select)
