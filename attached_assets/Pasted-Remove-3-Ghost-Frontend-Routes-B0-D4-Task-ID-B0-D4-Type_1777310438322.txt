Remove 3 Ghost Frontend Routes (B0-D4)
Task ID: B0-D4 Type: TDD Hardening Priority: MEDIUM — parallelizable with T4 Estimated effort: 1–2 hours Dependency: None
OBJECTIVE
Remove three frontend routes that exist in the navigation/router with no corresponding backend service: /dashboard/finance, /admin/loans, /investors. These are D4 drift artifacts — routes that were planned but never implemented, creating dead navigation links that confuse real users and surface in the admin UI.
PREFLIGHT CHECK
•          ☐ Locate the frontend router file (likely src/App.tsx or src/router.tsx)
•          ☐ Run grep -r "dashboard/finance\|admin/loans\|investors" src/ --include="*.tsx" --include="*.ts" — map all references
•          ☐ Confirm no backend route handlers exist for these paths
PERMISSION TO EXECUTE
Execute any time — independent of all other tasks. No backend changes required.
CONFLICT DETECTION
•          These routes may appear in: router config, nav sidebar/menu components, breadcrumb components, any link that references them
•          Check all three locations before deleting — a partial removal leaves broken links
•          Do NOT remove /investors if it refers to a real investor-facing page (verify it’s the dead route, not a legitimate page)
PROPOSED APPROACH
1.        Find all references to each of the three paths
2.        Remove route definitions from router
3.        Remove navigation links from sidebar/menu
4.        Confirm no 404 pages are introduced
ALTERNATIVES
Alternative	Reason Rejected
Redirect routes to /dashboard	Adds noise; dead features shouldn’t have redirects
Leave routes, hide from nav	Routes still accessible by URL — creates confusion
Add placeholder pages	Scope creep; these features are Phase 2+
