# FINCAVA.COM — Full-Stack Web App Build Prompt
## For Replit / Claude Code / Any AI Code Agent

---

## ROLE & CONTEXT

You are a senior full-stack engineer building **Fincava.com**, a B2B sourcing marketplace that connects verified Colombian agricultural producers (coffee, cacao, exotic fruits, superfoods, artisanal goods) directly to international buyers across the Middle East, Asia, and Africa. Think: Alibaba meets Faire, but exclusively for Colombian premium agricultural exports — with built-in trust, traceability, logistics coordination, and compliance tools.

Build a production-ready, fully functional web application. No placeholder content. No "coming soon" pages. Every feature must work.

---

## TECH STACK

**Frontend:**
- React 18 + TypeScript
- Tailwind CSS (utility-first styling)
- React Router v6 (client-side routing)
- React Query (server state management)
- Zustand (client state)
- Framer Motion (animations)
- Lucide React (icons)

**Backend:**
- Node.js + Express (REST API)
- PostgreSQL (primary database via Supabase or direct)
- Supabase (auth, storage, realtime)
- Prisma ORM
- JWT authentication

**Additional:**
- Stripe (payments + escrow simulation)
- SendGrid (transactional email)
- Cloudinary (image/document uploads)
- Resend (email notifications)

**Deploy target:** Replit (ensure all env vars are configurable via Secrets)

## DATABASE SCHEMA

Create these tables in PostgreSQL via Prisma:

```prisma
model User {
  id            String   @id @default(cuid())
  email         String   @unique
  passwordHash  String
  role          Role     // BUYER | SUPPLIER | ADMIN
  profile       Profile?
  company       Company?
  createdAt     DateTime @default(now())
  orders        Order[]
  messages      Message[]
  reviews       Review[]
}

model Profile {
  id          String  @id @default(cuid())
  userId      String  @unique
  user        User    @relation(fields: [userId], references: [id])
  firstName   String
  lastName    String
  phone       String?
  country     String
  language    String  @default("en")
  avatarUrl   String?
}

model Company {
  id              String    @id @default(cuid())
  userId          String    @unique
  user            User      @relation(fields: [userId], references: [id])
  name            String
  type            CompanyType  // COOPERATIVE | EXPORTER | SMALLHOLDER | IMPORTER | DISTRIBUTOR | ROASTER | MANUFACTURER
  country         String
  region          String?
  description     String
  logoUrl         String?
  website         String?
  verified        Boolean   @default(false)
  certifications  Certification[]
  products        Product[]
  documents       Document[]
  createdAt       DateTime  @default(now())
}

model Product {
  id              String    @id @default(cuid())
  companyId       String
  company         Company   @relation(fields: [companyId], references: [id])
  name            String
  category        ProductCategory  // COFFEE | CACAO | AVOCADO | EXOTIC_FRUIT | SUPERFOOD | PROCESSED | TEXTILE | OTHER
  subCategory     String?
  description     String
  origin          String    // Colombian region e.g. "Huila", "Santander", "Antioquia"
  altitude        String?   // e.g. "1600-2000 masl"
  process         String?   // e.g. "Washed", "Natural", "Honey"
  variety         String?   // e.g. "Castillo", "Geisha", "Criollo"
  minOrderKg      Float
  maxOrderKg      Float?
  pricePerKgUSD   Float
  availableKg     Float
  harvestSeason   String?
  images          String[]
  certifications  String[]  // ["Organic", "Fair Trade", "Rainforest Alliance"]
  cupping         Float?    // 0-100 SCA score for coffee
  active          Boolean   @default(true)
  featured        Boolean   @default(false)
  createdAt       DateTime  @default(now())
  orderItems      OrderItem[]
  reviews         Review[]
  inquiries       Inquiry[]
}

model Certification {
  id          String   @id @default(cuid())
  companyId   String
  company     Company  @relation(fields: [companyId], references: [id])
  type        String   // "USDA Organic" | "EU Organic" | "Fair Trade" | "Rainforest Alliance" | "UTZ" | "Halal" | "SCA Q Grade"
  issuer      String
  expiryDate  DateTime?
  documentUrl String?
  verified    Boolean  @default(false)
}

model Order {
  id              String      @id @default(cuid())
  buyerId         String
  buyer           User        @relation(fields: [buyerId], references: [id])
  status          OrderStatus // INQUIRY | SAMPLE_REQUESTED | QUOTED | CONFIRMED | IN_PRODUCTION | SHIPPED | DELIVERED | COMPLETED | CANCELLED
  totalUSD        Float
  incoterm        String      @default("FOB")
  destinationPort String?
  shippingMethod  String?     // "Sea Freight" | "Air Freight"
  notes           String?
  items           OrderItem[]
  documents       Document[]
  createdAt       DateTime    @default(now())
  updatedAt       DateTime    @updatedAt
}

model OrderItem {
  id          String   @id @default(cuid())
  orderId     String
  order       Order    @relation(fields: [orderId], references: [id])
  productId   String
  product     Product  @relation(fields: [productId], references: [id])
  quantityKg  Float
  pricePerKg  Float
  totalUSD    Float
}

model Inquiry {
  id          String   @id @default(cuid())
  productId   String
  product     Product  @relation(fields: [productId], references: [id])
  buyerEmail  String
  buyerName   String
  company     String
  country     String
  message     String
  quantityKg  Float?
  status      String   @default("PENDING")  // PENDING | RESPONDED | CLOSED
  createdAt   DateTime @default(now())
}

model Message {
  id          String   @id @default(cuid())
  senderId    String
  sender      User     @relation(fields: [senderId], references: [id])
  receiverId  String
  content     String
  read        Boolean  @default(false)
  createdAt   DateTime @default(now())
}

model Review {
  id          String   @id @default(cuid())
  authorId    String
  author      User     @relation(fields: [authorId], references: [id])
  productId   String
  product     Product  @relation(fields: [productId], references: [id])
  rating      Int      // 1-5
  comment     String?
  verified    Boolean  @default(false)
  createdAt   DateTime @default(now())
}

model Document {
  id          String   @id @default(cuid())
  companyId   String?
  orderId     String?
  company     Company? @relation(fields: [companyId], references: [id])
  order       Order?   @relation(fields: [orderId], references: [id])
  type        String   // "Certificate of Origin" | "Phytosanitary" | "Invoice" | "Packing List" | "Bill of Lading" | "Quality Report"
  name        String
  url         String
  createdAt   DateTime @default(now())
}

enum Role { BUYER SUPPLIER ADMIN }
enum CompanyType { COOPERATIVE EXPORTER SMALLHOLDER IMPORTER DISTRIBUTOR ROASTER MANUFACTURER }
enum ProductCategory { COFFEE CACAO AVOCADO EXOTIC_FRUIT SUPERFOOD PROCESSED TEXTILE OTHER }
enum OrderStatus { INQUIRY SAMPLE_REQUESTED QUOTED CONFIRMED IN_PRODUCTION SHIPPED DELIVERED COMPLETED CANCELLED }
```

## FEATURE SPECIFICATIONS

### Search & Filter System
- Full-text search across product name, description, supplier name, origin
- Real-time filter updates without page reload
- URL-based filter state (shareable links)
- "Clear all filters" button

### Inquiry & Sample Request Flow
1. Buyer clicks "Request Sample" or "Place Inquiry" on any product
2. If not logged in → redirect to login with return URL
3. If logged in → modal opens with pre-filled product info
4. Buyer fills in quantity, port, message
5. Submit → creates Inquiry record in DB
6. Email sent to supplier (SendGrid template)
7. Email confirmation sent to buyer
8. Both can see inquiry in their respective dashboards
9. Supplier can respond with quote (text + price + timeline)
10. Buyer can accept quote → creates Order

### Messaging System
- Real-time via Supabase Realtime subscriptions
- Message threads linked to Order or Inquiry
- Unread message count badge in nav
- Typing indicator
- File attachment (documents, images)

### Verification System (Supplier)
Suppliers start as "Pending". To become "Verified":
- [ ] Complete company profile
- [ ] Upload valid company registration
- [ ] Upload at least 1 product with full details
- [ ] Upload at least 1 certification
- Admin reviews and clicks "Verify"
- Verified badge appears on all their listings

### Wishlist / Save Products
- Heart icon on each product card
- Saved to buyer profile
- "Saved Products" section in dashboard

### Multi-language
- Language switcher: EN | ES | AR
- Use i18next for translations
- RTL support for Arabic
- Key pages translated: Homepage, Marketplace, Product Detail

### SEO
- React Helmet for meta tags
- Open Graph tags for product sharing
- Structured data (JSON-LD) for products
- Sitemap generation
- Clean URLs (no hash routing)

## WHAT MAKES FINCAVA DIFFERENT (build these into the UX)

1. **Every product has an origin story** — not just specs, but photos of the farm, the farmer's name, GPS coordinates of the region, seasonal availability calendar

2. **Compliance-ready documents** — every supplier upload includes certificate of origin, phytosanitary cert section; buyers can download these

3. **Regional market guides** — built-in pages showing what buyers in UAE, China, South Korea, and West Africa are specifically looking for

4. **Transparent pricing** — all prices in USD/kg, no hidden fees; platform takes 8-12% on confirmed orders, shown clearly

5. **Sample-first workflow** — the default CTA is always "Request Sample" not "Buy Now" — this matches how real B2B agricultural trade actually works

6. **Halal certification flag** — all products that are Halal certified show a prominent Halal badge (critical for UAE/Middle East market)

7. **Escrow language** — every order page shows "Payment secured by Fincava Escrow — released to supplier upon delivery confirmation"
