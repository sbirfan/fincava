CP-3 — Supplier product POST stores productTypeKey, typeAttributes,
        and sets productStatus='draft'
────────────────────────────────────────────────────────────────────────

You need a valid supplier auth token. Either:
  a) Use an existing test supplier JWT from .env (TEST_SUPPLIER_TOKEN), or
  b) Log in via POST /api/auth/login with test supplier credentials to get a token.

Store the token as SUPPLIER_TOKEN.

Run:
  RESPONSE=$(curl -s -X POST "$API_BASE_URL/api/supplier/products" \
    -H "Authorization: Bearer $SUPPLIER_TOKEN" \
    -H "Content-Type: application/json" \
    -d '{
      "name": "CP-3 Test Coffee",
      "category": "COFFEE",
      "productTypeKey": "COFFEE_GREEN",
      "origin": "Huila, Colombia",
      "pricePerKgUSD": 8.5,
      "minOrderKg": 100,
      "availableKg": 500,
      "typeAttributes": {
        "screen_size": "15",
        "moisture_content_pct": 11.5,
        "harvest_year": 2025,
        "drying_method": "Sun Dried"
      }
    }')

  echo $RESPONSE | jq '{
    id: .id,
    productTypeKey: .productTypeKey,
    typeAttributes: .typeAttributes,
    productStatus: .productStatus
  }'

PASS conditions:
  - .productTypeKey == "COFFEE_GREEN"
  - .typeAttributes.screen_size == "15" (or similar — confirms JSONB stored)
  - .productStatus == "draft"
  - .id is a positive integer

Store the product ID as PRODUCT_ID for use in CP-4 and CP-6:
  PRODUCT_ID=$(echo $RESPONSE | jq '.id')

Also confirm the data was written to the DB:
  psql $DATABASE_URL -c \
    "SELECT id, product_type_key, type_attributes->'screen_size' AS screen, product_status FROM products WHERE id = $PRODUCT_ID;"
