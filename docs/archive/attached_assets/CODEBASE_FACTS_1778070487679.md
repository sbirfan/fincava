## Auth System — CRITICAL (Confirmed from live repo)

**JWT-based tokens, HTTP-only cookie primary with Bearer fallback.**

```typescript
// requireAuth reads:
// 1. Cookie: req.cookies.fincava_auth (HTTP-only, 7-day expiry — primary)
// 2. Header: Authorization: Bearer <token> (fallback, still in code)
// Both are JWT tokens signed with JWT_SECRET

// requireAuth sets on req:
(req as any).userId   // number
(req as any).userRole // "BUYER" | "SUPPLIER" | "ADMIN"

// NOT req.user — does not exist
// NOT req.supplierId — does not exist
```

**Middleware:**
- `requireAuth` — `lib/auth.ts`
- `requireVerifiedEmail` — `lib/auth.ts` — chain AFTER requireAuth
- `requireAdmin` — `middleware/admin.ts`
- `adminOnly` — `[requireAuth, requireAdmin] as const` — use for all admin routes
- `requireRole(role)` — async factory, rarely used
- **`requireSupplier` — DOES NOT EXIST**

**Additional auth.ts exports (preserve on any edit):**
`getUserWithProfile(userId)`, `hashPassword(password)`, `verifyPassword(password, hash)`, `generateToken(userId)`, `verifyToken(token)`

## File Uploads — objectStorage.ts

```typescript
import { ObjectStorageService } from '../lib/objectStorage';
const objectStorageService = new ObjectStorageService();

// Get presigned upload URL — NO parameters:
const uploadURL = await objectStorageService.getObjectEntityUploadURL();
// Returns: string (presigned URL)

// Normalize to object path for storage:
const objectPath = objectStorageService.normalizeObjectEntityPath(uploadURL);

// Client flow:
// 1. Call POST /storage/uploads/request-url → { uploadURL, objectPath }
// 2. Client PUTs file directly to uploadURL
// 3. Client POSTs objectPath to compliance route
// 4. Compliance route stores objectPath in DB
```

**No multer. No file on the API server. Presigned URL handles all uploads client-side.**

Storage routes (no auth required on upload URL):
- `POST /storage/uploads/request-url` — body: `{ name, size, contentType }`
- `GET /storage/public-objects/*`
