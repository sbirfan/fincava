Admin Filter: Show Scored-But-Not-Evaluated Suppliers (G1-fix)
Task ID: G1-fix Type: TDD Hardening — Admin Workflow Priority: MEDIUM — parallelizable with T4 Estimated effort: 2–3 hours Dependency: None (complements T12/Dig, but independent)
OBJECTIVE
Add a filter to GET /api/admin/suppliers that returns suppliers who have an ai_outputs row (scored by AI) but no corresponding supplier_evaluations row (human evaluation not yet done). This is the admin’s daily action queue — the list of suppliers waiting for human review to graduate.
Without this filter, the admin must scroll through all suppliers to find which ones need attention. As supplier count grows, this becomes operationally unworkable.
PREFLIGHT CHECK
•          ☐ Read routes/admin.ts or routes/suppliers.ts — find GET /api/admin/suppliers
•          ☐ Confirm ai_outputs table has supplier_id FK
•          ☐ Confirm supplier_evaluations table has supplier_id FK
•          ☐ Test current admin supplier endpoint — note existing filter params
PERMISSION TO EXECUTE
Execute any time. Additive change — existing admin functionality unchanged.
CONFLICT DETECTION
•          T12 (Dig — daily digest email) also queries scored-but-not-evaluated suppliers. T10 adds the API filter; T12 adds the scheduled email using the same logic. Do not duplicate the query — extract to a shared service function if building T10 and T12 together.
•          No conflict with T4 (AI1) — this task doesn’t touch scoring logic
PROPOSED APPROACH
Add ?filter=scored_not_evaluated query parameter to the existing admin suppliers endpoint. The query: SELECT s.* FROM suppliers s INNER JOIN ai_outputs ao ON ao.supplier_id = s.id LEFT JOIN supplier_evaluations se ON se.supplier_id = s.id WHERE se.id IS NULL.
ALTERNATIVES
Alternative	Reason Rejected

Frontend-side filtering	Requires fetching all suppliers; doesn’t scale
Defer to T12 email	T12 notifies; T10 enables the admin UI workflow — both needed
