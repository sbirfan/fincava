## 1. POST /api/admin/suppliers/:id/transition

Purpose:
Manual state transition (admin override)

---

### Request body

{
"toState": "NOT_READY | ELIGIBLE | SELLABLE | PUBLISHED",
"actor": "ADMIN | FOUNDER",
"justification": "string (required)"
}

---

### Behavior

Call:

transitionTo(
supplierId,
toState,
actor,
{
justification
}
)

---

### Validation

* justification REQUIRED for ADMIN and FOUNDER
* actor must NOT be SYSTEM (reject if provided)
* toState must be valid enum

---

### Response

{
"transition": { ...transitionRow }
}

---

## 2. POST /api/admin/suppliers/:id/publish

Purpose:
Explicit publish action (EP3 enforcement)

---

### Request body

{
"actor": "ADMIN | FOUNDER",
"justification": "string (required)"
}

---

### Behavior

Call:

markPublished(
supplierId,
actor,
justification
)

---

### Additional validation (CRITICAL — 1.10)

Before calling markPublished:

1. Fetch supplier row
2. Verify:

* supplier.sellableStatus === "SELLABLE"

IF NOT:

→ return 400

{
"error": "Supplier must be SELLABLE before publishing"
}

---

### Response

{
"transition": { ...transitionRow }
}
